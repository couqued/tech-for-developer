Kubernetes 컨테이너 접속
``` bash
$ kubectl exec -it pod/<pod-name> -n <namespace-name> -- sh
```
<br />

Kubernetes pod db 접속
``` bash
psql -h localhost -U <user-name> -d <db-name>
```
<br />

Kubernetes 약어 보기
``` bash
kubectl api-resources
```
<br />

Node 상세 내용 보기
``` bash
kubectl describe node <Node-name>
```
<br />

Namespace의 서비스 조회
``` bash
kubectl get svc --namespace default
kubectl get svc -n kube-system
```
<br />

모든 디플로이먼트 삭제 (pod도 함께 삭제됨)
```bash
kubectl delete deploy --all
```
<br />

모든 서비스 삭제 (default 네임스페이스에 있는 쿠버네티스 API까지 삭제됨)
```bash
kubectl delete svc --all
```
<br />

남아있는 리소스 확인
```bash
kubectl get all
```
<br />

pod가 준비될때까지 대기
```bash
kubectl wait --for=condition=Ready pod -l app=sleep
```
<br />

pod 속 컨테이너에 설정된 환경 변수 확인 (sleep이라는 디플로이먼트의 HOSTNAME과 환경변수 값 출력)
```bash
kubectl exec deploy/sleep -- printenv HOSTNAME <환경변수 name>
```

```bash
spec:
  container:
    - name: sleep
      image: sample/sleep
      env: 
      - name: VAR_NAME # 환경변수 이름
        value: “04”    # 환경변수 값
```
